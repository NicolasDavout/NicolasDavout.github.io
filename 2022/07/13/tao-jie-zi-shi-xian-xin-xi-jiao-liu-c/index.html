套接字

在学习之前需注意一下几点：

  • 若不了解自行了解套接字相关概念及理论
  • 此内容均基于windows系统
  • 标题带*号的表示这部分内容不太重要，可以跳过，代码直接照抄

C++中使用套接字的流程如下：

我们首先放上服务器和客户端的代码

代码

服务器代码

#include <stdio.h>
#include <winsock2.h>
#pragma comment (lib, "ws2_32.lib")  //加载 ws2_32.dll
int main() {
    //初始化 DLL
    WSADATA wsaData;
    WSAStartup(MAKEWORD(2, 2), &wsaData);
    //创建套接字
    SOCKET servSock = socket(AF_INET, SOCK_STREAM, IPPROTO_TCP);
    //绑定套接字
    sockaddr_in sockAddr;
    memset(&sockAddr, 0, sizeof(sockAddr));  //每个字节都用0填充
    sockAddr.sin_family = PF_INET;  //使用IPv4地址
    sockAddr.sin_port = htons(1234);  //端口
    bind(servSock, (SOCKADDR*)&sockAddr, sizeof(SOCKADDR));
    //进入监听状态
    listen(servSock, 20);
    //接收客户端请求
    SOCKADDR clntAddr;
    int nSize = sizeof(SOCKADDR);
    SOCKET clntSock = accept(servSock, &clntAddr, &nSize);
    //向客户端发送数据
    const char* str = "Hello World!";
    send(clntSock, str, strlen(str) + sizeof(char), NULL);
    //关闭套接字
    closesocket(clntSock);
    closesocket(servSock);
    //终止 DLL 的使用
    WSACleanup();
    return 0;
}

客户端代码

#define _WINSOCK_DEPRECATED_NO_WARNINGS
#include <stdio.h>
#include <stdlib.h>
#include <WinSock2.h>
#pragma comment(lib, "ws2_32.lib")  //加载 ws2_32.dll

int main() {
    //初始化DLL
    WSADATA wsaData;
    WSAStartup(MAKEWORD(2, 2), &wsaData);

    //创建套接字
    SOCKET sock = socket(AF_INET, SOCK_STREAM, IPPROTO_TCP);

    //向服务器发起请求
    sockaddr_in sockAddr;
    memset(&sockAddr, 0, sizeof(sockAddr));  //每个字节都用0填充
    sockAddr.sin_family = PF_INET;
    sockAddr.sin_addr.s_addr = inet_addr("127.0.0.1");
    sockAddr.sin_port = htons(1234);
    connect(sock, (SOCKADDR*)&sockAddr, sizeof(SOCKADDR));

    //接收服务器传回的数据
    char szBuffer[MAXBYTE] = { 0 };
    recv(sock, szBuffer, MAXBYTE, NULL);

    //输出接收到的数据
    printf("Message form server: %s\n", szBuffer);

    //关闭套接字
    closesocket(sock);

    //终止使用 DLL
    WSACleanup();

    system("pause");
    return 0;
}

代码解析

关于客户端的第一行的define*

客户端第一行代码

//客户端
#define _WINSOCK_DEPRECATED_NO_WARNINGS

这个是防止报错的，如果在VS中使用了比较老旧落后的函数，就会报错

编译器会进行SDL检查

安全开发生命周期（SDL）

在SDL检查中发现了inet_addr()这个落后的函数，于是报错，不予通过

inet_addr()是落后的函数，因为它仅仅支持ipv4

所以为了能够强行通过编译，我们直接取消掉SDL检查

所以在首行就写

#define _WINSOCK_DEPRECATED_NO_WARNINGS

inet_addr()的功能是将一个点分十进制的IP转换成一个长整数型数（u_long类型）

inet_addr()头文件与函数原型

#include <WinSock2.h>
in_addr_t inet_addr(const char *cp);

函数里面填ipv4地址

随着ipv6地址的普及inet_addr()函数已经基本废弃，新的函数inet_pton()能够代替之

inet_pton是一个IP地址转换函数，可以在将点分文本的IP地址转换为二进制网络字节序”的IP地址，而且inet_pton能够处理ipv4和ipv6

inet_pton()头文件与函数原型

#include <WS2tcpip.h>
int inet_pton(int af, const char *src, void *dst);

第一个参数af，这里提供三个比较常用的值

  • AF_INET (或者PF_INET) 通过ipv4地址访问(同样可以访问本机)
  • AF_INET6 (或者PF_INET6) 通过ipv6地址访问
  • PF_UNIX (或者PF_LOCAL) 访问本地(127.0.0.1)

第二个参数src即ipv4地址或ipv6地址或者本机地址(127.0.0.1)

如果参数af指定的地址族和src格式不对，函数将返回0

第三个参数填入套接字存储ip地址的内存地址

这里关于套接字的结构后面再说，在此先做点了解

所以如果想要支持ipv6的话就不是写取消SDL检查这么简单，步骤如下：

  • 删除首行 #define _WINSOCK_DEPRECATED_NO_WARNINGS
  • 包含头文件 #include <WS2tcpip.h>
  • 将旧版函数sockAddr.sin_addr.s_addr = inet_addr(“127.0.0.1”)改为新版函数inet_pton(AF_INET,”127.0.0.1”, &sockAddr.sin_addr)

头文件与动态链接库

  • #include<stdio.h>

标准输入输出头文件

  • #include<stdlib.h>

很多转换函数都在此头文件中

  • #include<winsock2.h>

套接字必备头文件

  • #include <WS2tcpip.h>

inet_pton()函数所在头文件

  • #pragma comment (lib, “ws2_32.lib”)

链接Ws2_32.lib这个库，加载 ws2_32.dll

动态链接库 Dynamic Link Library （DLL）

启动套接字*

然后我们启动套接字，启动函数是

#include <WinSock2.h>
int WSAStartup( WORD wVersionRequested, LPWSADATA lpWSAData);

第一个参数的含义是版本

第一个参数的类型是WORD

WORD是一个双字节型数值，就是把两个字节拼接在一起

使用MAKEWORD函数把两个数组成WORD类数据

MAKEWORD(a,b) 最终结果是b在高位，a在低位

比如MAKEWORD(2,1) 00000010(a) 00000001(b)

最终的结果是 00000001 00000010

MAKEWORD(2,2)的意思就是当前使用2.2版本

第二个参数就是一个一个指针，指向WSADATA类型的数据

WSADATA，一种数据结构。这个结构被用来存储被WSAStartup函数调用后返回的Windows Sockets数据。它包含Winsock.dll 执行的数据。

通过WSAStartup函数启动，再用WSADATA类的数据把启动之后的相关执行数据储存。

//服务器
WSADATA wsaData;
WSAStartup(MAKEWORD(2, 2), &wsaData);

这两句就这么个意思：

开一个WSADATA的对象wsaData存储启动数据

以2.2版本的Windows Sockets启动套接字，并且把启动数据传给对象wsaData

创建套接字

//服务器
SOCKET servSock = socket(AF_INET, SOCK_STREAM, IPPROTO_TCP);

socket函数原型

SOCKET socket(int af, int type, int protocol);

af上文有讲

  • ipv4 af=AF_INET
  • ipv6 af=AF_INET6

后两个个参数具有绑定的关系，一般有以下两种情况：

  • type=SOCK_STREAM protocol=IPPROTO_TCP
  • type=SOCK_DGRAM protocol=IPPROTO_UDP

第一种传输可靠性高（进行校验，并能够重新发送），但速度慢
第二种传输可靠性低（不校验，只发送一次），但速度快

一般对速度无要求都用第一种，需要实时传输的比如视频通话就要使用第二种

protocol也可以填写0，会根据type自动匹配protocol

创建套接字地址

创建一个套接字的对象

//服务器
sockaddr_in sockAddr;

sockaddr_in是一个结构，其成员如下：

struct socketaddr_in
{
    sa_family_t sin_family;//地址族
    unit16_t sin port;//16位TCP/UDP端口号
    struct in_addr sin_addr;// 32位IP地址
    char sin-zero[8];//不使用
};

char sin-zero[8]是为了让SOCKADDR与sockaddr_in两个数据结构保持大小相同而保留的空字节（SOCKADDR后面bind函数要用）

其中的in_addr又是一个结构，该结构只有一个成员

struct in_addr
{
    In_addr_t s_addr;//32位ipv4地址
};

所以sockaddr_in sockAddr意思就是创建套接字，用于存储服务器地址、端口

服务器套接字的初始化

memset将某一块内存中的内容全部设置为指定的值， 这个函数通常为新申请的内存做初始化工作。

memset(&sockAddr, 0, sizeof(sockAddr)); //每个字节都用0填充

void *memset(void *s, int ch, size_t n);

void *s 代表无类型指针，可以指向任意的数据地址

用n个字节的ch字符填充s指向地址为起点的区域

//服务器
memset(&sockAddr, 0, sizeof(sockAddr));  //每个字节都用0填充

这个意思就是把套接字地址结构的对象(sockAddr)内存全部初始化为0

初始化完成之后开始具体内容的赋值

//服务器
sockAddr.sin_family = PF_INET;

前文有讲，地址族主要有以下三个参数

  • AF_INET (或者PF_INET) 通过ipv4地址访问(同样可以访问本机)
  • AF_INET6 (或者PF_INET6) 通过ipv6地址访问
  • PF_UNIX (或者PF_LOCAL) 访问本地(127.0.0.1)

由于服务器是接收客户端的地址，所以无需在服务器设置某个地址

关于

//服务器
sockAddr.sin_port = htons(1234);

设置端口为1234

uint16_t 的长度为两个字节，理论上端口号的取值范围为 065536，但 01023 的端口一般由系统分配给特定的服务程序，例如 Web 服务的端口号为 80，FTP 服务的端口号为 21，所以我们的程序要尽量在 1024~65536 之间分配端口号

htons函数将整型变量从主机字节顺序转变成网络字节顺序

网络字节顺序是TCP/IP中规定好的一种数据表示格式，它与具体的CPU类型、操作系统等无关，从而可以保证数据在不同主机之间传输时能够被正确解释

绑定端口和socket

上面的sock（套接字）和sockAddr（套接字地址）是分开定义的，需要把他们绑定

使用bind函数进行绑定

int bind(SOCKET sock, const struct sockaddr *addr, int addrlen);

前面创建了两个东西：套接字servSock和套接字地址sockAddr，对应第一个和第二个参数分别传入

bind(servSock, (SOCKADDR*)&sockAddr, sizeof(SOCKADDR));

第二个参数注意把sockaddr_in强制转换为SOCKADDR类型

sockaddr的缺陷是：sa_data把目标地址和端口信息混在一起了

sockaddr_in解决了sockaddr的缺陷

第三个参数是地址的长度，直接使用sizeof函数取大小

servSock和sockAddr绑定完成

服务器进入监听状态

监听状态使用listen函数

int listen(SOCKET sockfd, int backlog);

第一个参数传入绑定之后的套接字

第二个是指定队列的长度，在队列中的客户端请求接受，在队列外的拒绝（长度与接受客户端请求数量正相关）

//服务器
listen(servSock, 20);

这里进行监听并且储存在绑定之后的套接字servSock中，指定队列的长度为20

获取客户端地址

//服务器
SOCKADDR clntAddr;

创建一个套接字地址结构的对象clntAddr，把客户端地址相关信息存储进去

接受客户端信息

接受信息使用accept函数

原型如下

SOCKET accept(SOCKET sock, struct sockaddr *addr, int *addrlen);

第一个参数sock就是服务器创建的套接字

当程序运行到accept函数时，程序会阻塞，不会继续运行，直到有客户端的请求时才会继续运行

当accepot函数接受到了客户端的请求，会把客户端地址的相关信息存储在addr指向的内存中

所以第二个参数就是传入一个套接字地址SOACADDR指针用于存储

第三个参数是地址的长度，直接把SOACADDR的长度传入即可

//服务器
int nSize = sizeof(SOCKADDR);//SOCKADDR长度
SOCKET clntSock = accept(servSock, &clntAddr, &nSize);

新建一个套接字对象clntSock，给它通过accept函数加入服务器套接字信息servSock和客户端地址clntAddr

也就是说clntSock具备了服务器套接字和客户端地址的信息，目前就剩一个发送内容

给客户端发送消息

发送消息使用send函数

//服务器
int send(SOCKET sock, const char *buf, int len, int flags);

第一个参数是客户端的套接字地址，第二个参数是发送的内容，第三个是发送的长度，第四个flag是数据选项，一般设置为0或NULL

//服务器
send(clntSock, str, strlen(str) + sizeof(char), NULL);

意思就是通过客户端套接字把str字符串中str长度+1的内容发送给客户端

str长度+1是把结束符“\0”也发送，避免不必要的错误

关闭套接字

使用closesocket函数

直接把要关闭的套接字当参数传入即可

//服务器
closesocket(clntSock);
closesocket(servSock);

这里把客户端和服务器套接字都关闭

注意：这个客户端套接字是在服务器创建的（理解为面向客户端的在服务器的套接字），和客户端上的套接字独立

终止DLL的调用

//服务器
WSACleanup();

DLL启用后也要关闭，用WSACleanup函数终止DLL的调用

客户端在接收之前的操作

//客户端
    //初始化DLL
    WSADATA wsaData;
    WSAStartup(MAKEWORD(2, 2), &wsaData);

    //创建套接字
    SOCKET sock = socket(AF_INET, SOCK_STREAM, IPPROTO_TCP);

    //向服务器发起请求
    sockaddr_in sockAddr;
    memset(&sockAddr, 0, sizeof(sockAddr));  //每个字节都用0填充
    sockAddr.sin_family = PF_INET;//指定为ipv4地址类型
    sockAddr.sin_addr.s_addr = inet_addr("127.0.0.1");//填写ipv4地址
    sockAddr.sin_port = htons(1234);//填写端口

和服务器一样

客户端比服务器多填写一个ip地址

和服务器建立连接

使用connect函数

int connect(SOCKET sock, const struct sockaddr *serv_addr, int addrlen);

//客户端
connect(sock, (SOCKADDR*)&sockAddr, sizeof(SOCKADDR));

connect函数用于和服务器建立链接，各个参数和bind函数一样

注意：connect函数和bind函数的区别

  • connect用于远程连接（填写本机地址也可以）
  • bind用于本地连接

接受服务器返回数据

使用recv函数接收服务器返回数据

int recv(SOCKET s, char* buf, int len, int flags);

receive函数和send函数参数同理

//客户端
char szBuffer[MAXBYTE] = { 0 };
recv(sock, szBuffer, MAXBYTE, NULL);

注意MAXBYTE的定义

#define MAXBYTE 0xff

MAXBYTE就是十进制的255

这段意思就是开个255大小的char数组全部初始化为0

然后再从服务器接受字符填入szBuffer所指向的数组中

接受完服务器数据之后

//关闭套接字
    closesocket(clntSock);
    closesocket(servSock);
//终止 DLL 的使用
    WSACleanup();

和服务器一样
